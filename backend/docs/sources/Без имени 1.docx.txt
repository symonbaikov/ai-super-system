### A. Память/контекст системы
#### Нарратив-библия (RAG)
Это Retrieval-Augmented Generation (RAG) база: структурированный список нарративов для подмешивания в каждый анализ токена. Подмешивать автоматически: перед анализом добавлять релевантные нарративы из базы для контекста (e.g., "Учитывая нарратив AI, проверь пересечения"). Обоснование: Увеличивает точность анализа, связывая токен с трендами, снижая риски FOMO/хайпа. База обновляется по поиску на X/Web.
**Список тем (Narratives):**
- AI: Токены с ИИ-тематикой (e.g., Grok, ChatGPT-inspired). Сила: Высокая, если связан с xAI/Elon. Синонимы: artificial intelligence, neural nets.
- Politics: Политические мемы (e.g., Trump, Biden parodies). Сила: Пиковая во время выборов (2024-2025). Синонимы: MAGA, election.
- Animals: Животные мемы (e.g., dogs, cats, frogs). Сила: Вечнозеленая, хайп от Dogecoin. Синонимы: pupper, kitty.
- Elon Musk: Токены, связанные с Musk (e.g., Tesla, SpaceX). Сила: Взрывная от твитов. Синонимы: Doge father, Mars.
- Gaming/NFT: Игры/NFT (e.g., metaverse). Сила: Средняя, зависит от bull market.
- DeFi/Finance: Финансовые инструменты. Сила: Низкая для мемов, но стабильная.

#### Ассистент-Ресерчер (GPT)
Роль: Исследователь мем-коинов Solana. Обоснование: Автоматизирует due diligence, минимизируя риски (e.g., rugs). Чек-лист: Стандартизированный для объективности.
**Чек-лист проверки:**
- LP (Liquidity Pool): ≥10k USD – минимальный пул ликвидности для снижения манипуляции. Значение: Объем заблокированных средств в паре (e.g., TOKEN/SOL). Низкий LP = риск rug pull.
- Owner% (Владение создателем): ≤5% (X=5) – процент токенов у owner wallet. Высокий = риск dump.
- Sniper count: ≤10 – количество ранних покупателей, продающих быстро (snipe bots). Высокий = orchestrated pump/dump.
- Burn/Mint/Blacklist: Burned LP (сожженный) = да; Mint authority revoked = да; Blacklist addresses = нет. Burn: Сожжение токенов для дефляции. Mint: Возможность допечатки = риск инфляции. Blacklist: Заблокированные адреса = потенциальный scam.
- Холдеры-“красные” (Red holders): Нет подозрительных (e.g., wallets с историей scams, multiple rugs via Solscan). Красные: Флагированные как malicious (e.g., high txn count, quick sells).
**Формат вывода:**
Таблица решений: Колонки [Параметр, Значение, Риск (Low/Med/High), Причина].
Вердикт: BUY (низкий риск), WAIT (средний, мониторить), NO-GO (высокий, избегать).
### B. Стандартизованные промпты
#### 1. Аудит мема (короткий чек-лист)
Промпт: «Ты — аудит-бот мемов Solana. Дано: {DexScreener/GMGN сводка + скриншот/таблица}. Проверь: LP, market cap, owner%, топ-холдеры, sniper-продажи, burn/mint/blacklist, активность твитов за 60 мин. Вывод: таблица полей [Риск, Причина, Действие, Срочность], итоговый вердикт: BUY/WAIT/NO-GO. Сначала уточняющие вопросы (макс 3), если данных не хватает.»
Обоснование: Быстрый аудит для входа. Термины: Market cap = рыночная капитализация (total supply * price). Sniper-продажи = продажи snipers. Активность твитов = посты/ретвиты за час (via X search).
#### 2. Sizing на старте по залочку SOL
Промпт: «Дано: creator locked {X} SOL, ликвидность {L}, спред {S}, волатильность {V}. Правила MM: риск на сделку {r%}, макс просадка дня {d%}. Посчитай стартовый объём входа (SOL) для стратегий: скальп (1–5 мин), краткосрок (час), с указанием SL/TP/трейлинга и частичных выходов (1x/2x). Формат: таблица.»
Обоснование: Money Management (MM) для контроля рисков. Термины: Locked SOL = заблокированный создателем (доверие). Спред = bid/ask разница. Волатильность V = std dev цены. Риск r% = % портфеля на trade. Просадка d% = max daily loss. SL = Stop Loss (цена выхода в убыток). TP = Take Profit. Трейлинг = trailing stop (динамический SL). Частичные выходы: Продавать порциями на 1x/2x прибыли.
#### 3. Твиттер-сигналы → решалка
Промпт: «Вход: {скрин/текст твитов}. Классифицируй: хайп/флуд/сигнал. Оцени силу нарратива (0–5), пересечения с трендами (Илон/политика/животные/AI и т.п.). Итог: стоит ли мониторить New Creations/Trending; какой фильтр в GMGN включить. Короткий план действий на 30–60 минут.»
Обоснование: Фильтр сигналов для timely actions. Термины: Хайп = эмоциональный buzz. Флуд = спам. Сигнал = actionable info. Сила 0-5: 0=слабый, 5=вирусный. Пересечения: С нарративами из библии. GMGN = GMGN.ai (Solana scanner). План: e.g., "Мониторь trending, фильтр LP>10k".

### C. Агент-пакет (минимум 4 GPT-ассистента)
Агент-пакет: Система из 4+ специализированных GPT-ассистентов для автоматизированного трейдинга мем-коинами на Solana. Обоснование: Разделение ролей повышает эффективность, снижает ошибки, позволяет параллельную обработку (e.g., scout → auditor → sizer). Каждый ассистент — промпт-ориентированный, интегрируется с RAG (нарратив-библия) для контекста. Минимально 4, но scalable.
#### 1. Narrative Scout
Роль: Анализирует твиты/скрины с X, присваивает "оценку хайпа" (0-10), сохраняет в вектор-память (embeddings для поиска). Обоснование: Раннее выявление трендов минимизирует FOMO, фокусируется на вирусности. 
- Оценка хайпа: 0=нет интереса (низкие лайки/ретвиты), 10=вирусный (хайп от инфлюенсеров, пересечения с нарративами как Elon/AI). Рассчитывается: (ретвиты + лайки)/время + сила нарратива (из библии).
- Вектор-память: Embeddings (via GPT) для хранения: тема, сила, timestamp. Доступ: Поиск по similarity для будущих запросов.
- Промпт: "Вход: {твиты/скрины}. Классифицируй нарратив, оцени хайп (0-10), добавь в память: [вектор: тема=AI, сила=8, ts=2025-09-16]."
#### 2. Token Auditor
Роль: Проводит ончейн-аудит по чек-листу + анти-rug правилам. Обоснование: Защищает от scams (rugs, honeypots), используя данные DexScreener/Solscan. Интегрирует с RAG для нарратив-чеков.
- Чек-лист: LP ≥10k USD (ликвидность для стабильности), owner% ≤5% (низкий контроль создателя), sniper count ≤10 (мало ботов для dump), burn=да (сожженные токены для дефляции), mint revoked=да (нет допечатки), blacklist=нет (нет заблокированных адресов), холдеры-красные=0 (нет scam-wallets).
- Анти-rug: Проверка renounce ownership (отказ от контроля), frozen accounts (заморозка для защиты).
- Вывод: Таблица [Параметр, Значение, Риск (Low/Med/High), Вердикт]. Итог: BUY/WAIT/NO-GO.
- Промпт: Из B.1, плюс "Интегрируй анти-rug: renounce=да, frozen=нет."
#### 3. Sizer/Breaker
Роль: Рассчитывает размер позиции, сетку TP/SL/трейлинг по профилю ликвидности. Обоснование: Контролирует риски в волатильном рынке, адаптирует под стратегии (скальп/краткосроч). Профиль ликвидности: LP размер, спред (bid/ask разница), волатильность (std dev цены).
- Размер позиции: Риск на trade r% от портфеля (e.g., 1-2%), макс просадка дня d% (e.g., 5%). Формула: Volume = (портфель * r%) / (SL расстояние).
- Сетка: TP (take profit) на 1x/2x/3x прибыли (частичное закрытие), SL (stop loss) на -10-20%, трейлинг (динамический SL, e.g., 5% от пика).
- Стратегии: Скальп (1-5 мин: tight SL/TP), краткосроч (час: wider, с трейлингом).
- Вывод: Таблица [Стратегия, Вход SOL, SL%, TP levels, Трейлинг].
- Промпт: Из B.2, плюс "Адаптируй по профилю: low LP=tight SL."
#### 4. Post-Trade Analyst
Роль: Анализирует после сделки: вход/выход, что сработало/нет, обновляет правила. Обоснование: Улучшает стратегии через feedback loop, повышает hit-rate (процент прибыльных трейдов).
- Разбор: Вход (цена/time), выход (причина: TP/SL/hit), PNL (profit/loss), факторы (хайп сработал? Rug avoided?).
- Обновляет: Правила в RAG (e.g., "Избегай owner%>3% после 5 rugs"), чек-листы (добавить новый параметр).
- Вывод: Отчет [Что сработало, Ошибки, Уроки, Обновленные правила].
- Промпт: "Вход: {лог трейда}. Разбери: вход/выход, PNL. Обнови правила в памяти."
### D. Автоматизация
Автоматизация: Интеграция с Google Sheets и Canvas для workflow. Обоснование: Централизует данные, автоматизирует сигналы/аудиты, позволяет live-обновления. Sheets для data-driven, Canvas для визуальных плейбуков.
#### Google Sheets
- Вкладка Signals: Колонки [Линки/адреса (твиты/Dex links), Оценка хайпа (от Scout)]. Формулы: =GPT("Классифицируй: {cell}", prompt из B.3) для авто-классификации (хайп/флуд/сигнал). Обоснование: Авто-фильтр сигналов, экспорт в память.
- Вкладка Audit: Колонки [LP, owner%, burn/mint, etc.]. GPT: =GPT("Аудит: {data}", prompt из B.1) расставляет риски/вердикт (цветовые сигналы: green=low, red=high). Обоснование: Быстрый скрининг, интеграция с Auditor.
- PNL & Hit-rate: Колонки [Трейд лог, PNL, Hit-rate (win/loss ratio)]. GPT: =GPT("Анализ: {log}", prompt от Analyst) генерит комментарии (e.g., "Hit-rate 70%, улучшить SL"). Обоснование: Track performance, feedback в RAG.
#### Canvas (4o Canvas)
- Live-редактирование плейбуков: Визуальные доски для стоп-лосс политики (e.g., SL= -15% для low LP), шаблоны размерности (e.g., r%=1% скальп), чек-листы входа (из Auditor). Обоснование: Коллаборативно, легко обновлять (drag-drop), экспорт в промпты. Интеграция: Экспорт как JSON в память ИИ.
### E. Голос/мобайл
Голос/мобайл: Voice-режим в Grok apps как "дежурный риск-менеджер". Обоснование: Hands-free для мобильных трейдеров, снижает импульсивные ошибки через verbal checklists. Доступ: Grok iOS/Android apps (voice mode для Grok 3+).
- Функции: 
  - "Повтори план сделки": Озвучивает sizing/TP/SL от Sizer (e.g., "Вход 5 SOL, SL -10%, TP 2x").
  - "Прочитай чек-лист перед входом": Verbal аудит от Auditor (e.g., "LP 15k: low risk; owner 2%: ok").
  - "Сделай сводку твитов за час": Summary от Scout (e.g., "Хайп 7/10, нарратив AI, 50 ретвитов").
- Интеграция: Voice input → текст промпт → ответ voice. Обоснование: Реал-тайм риски, для on-the-go (e.g., во время commute).
